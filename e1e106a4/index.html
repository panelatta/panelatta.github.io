A. Favorite Sequence (800)

Description

Time Limit: 2 seconds
Memory Limit: 256 megabytes

Polycarp has a favorite sequence a[1…n]a[1 \dots n]a[1…n] consisting of nnn integers. He wrote it out on the whiteboard as follows:

  • he wrote the number a1a_1a1​ to the left side (at the beginning of the whiteboard);
  • he wrote the number a2a_2a2​ to the right side (at the end of the whiteboard);
  • then as far to the left as possible (but to the right from a1a_1a1​), he wrote the number a3a_3a3​;
  • then as far to the right as possible (but to the left from a2a_2a2​), he wrote the number a4a_4a4​;
  • Polycarp continued to act as well, until he wrote out the entire sequence on the whiteboard.
The beginning of the result looks like this (of course, if n≥4n \ge 4n≥4).

For example, if n=7n=7n=7 and a=[3,1,4,1,5,9,2]a=[3, 1, 4, 1, 5, 9, 2]a=[3,1,4,1,5,9,2], then Polycarp will write a sequence on the whiteboard [3,4,5,2,9,1,1][3, 4, 5, 2, 9, 1, 1][3,4,5,2,9,1,1].

You saw the sequence written on the whiteboard and now you want to restore Polycarp’s favorite sequence.

B. Last Year’s Substring (800)

Description

Time Limit: 2 seconds
Memory Limit: 256 megabytes

Polycarp has a string s[1…n]s[1 \dots n]s[1…n] of length nnn consisting of decimal digits. Polycarp performs the following operation with the string sss no more than once (i.e. he can perform operation 000 or 111 time):

  • Polycarp selects two numbers iii and jjj (1≤i≤j≤n1 \leq i \leq j \leq n1≤i≤j≤n) and removes characters from the sss string at the positions i,i+1,i+2,…,ji, i+1, i+2, \ldots, ji,i+1,i+2,…,j (i.e. removes substring s[i…j]s[i \dots j]s[i…j]). More formally, Polycarp turns the string sss into the string s1s2…si−1sj+1sj+2…sns_1 s_2 \ldots s_{i-1} s_{j+1} s_{j+2} \ldots s_{n}s1​s2​…si−1​sj+1​sj+2​…sn​.

For example, the string s=s=s="20192020{\tt 20192020}20192020"​ Polycarp can turn into strings:

  • “2020\tt 20202020” (in this case (i,j)=(3,6)(i,j)=(3,6)(i,j)=(3,6) or (i,j)=(1,4)(i,j)=(1,4)(i,j)=(1,4));
  • “2019220\tt 20192202019220” (in this case (i,j)=(6,6)(i,j)=(6,6)(i,j)=(6,6));
  • “020\tt 020020” (in this case (i,j)=(1,5)(i,j)=(1,5)(i,j)=(1,5));
  • other operations are also possible, only a few of them are listed above.

Polycarp likes the string “2020\tt 20202020” very much, so he is wondering if it is possible to turn the string s into a string “2020\tt 20202020” in no more than one operation? Note that you can perform zero operations.

Input

The first line contains a positive integer ttt ($1 \leq t \leq 1000 $) — number of test cases in the test. Then ttt test cases follow.

The first line of each test case contains an integer nnn (4≤n≤2004 \leq n \leq 2004≤n≤200) — length of the string sss. The next line contains a string sss of length nnn consisting of decimal digits. It is allowed that the string sss starts with digit 0\tt 00.

Output

For each test case, output on a separate line:

  • “YES\tt YESYES” if Polycarp can turn the string sss into a string “2020\tt 20202020” in no more than one operation (i.e. he can perform 000 or 111 operation);
  • “NO\tt NONO” otherwise.

You may print every letter of “YES\tt YESYES” and “NO\tt NONO” in any case you want (so, for example, the strings yEs\tt yEsyEs, yes\tt yesyes, Yes\tt YesYes and YES\tt YESYES will all be recognized as positive answer).

D. Add to Neighbour and Remove (1400)

Description

Time Limit: 3 seconds
Memory Limit: 256 megabytes

Polycarp was given an array of a[1…n]a[1 \dots n]a[1…n] of nnn integers. He can perform the following operation with the array aaa no more than nnn times:

  • Polycarp selects the index iii and adds the value aia_iai​ to one of his choice of its neighbors. More formally, Polycarp adds the value of aia_iai​ to ai−1a_{i−1}ai−1​ or to ai+1a_{i+1}ai+1​ (if such a neighbor does not exist, then it is impossible to add to it).
  • After adding it, Polycarp removes the iii-th element from the aaa array. During this step the length of aaa is decreased by 111.

The two items above together denote one single operation.

For example, if Polycarp has an array a=[3,1,6,6,2]a = [3, 1, 6, 6, 2]a=[3,1,6,6,2], then it can perform the following sequence of operations with it:

  • Polycarp selects i=2i = 2i=2 and adds the value aia_iai​ to (i−1)(i-1)(i−1)-th element: a=[4,6,6,2]a = [4, 6, 6, 2]a=[4,6,6,2].
  • Polycarp selects i=1i = 1i=1 and adds the value aia_iai​ to (i+1)(i+1)(i+1)-th element: a=[10,6,2]a = [10, 6, 2]a=[10,6,2].
  • Polycarp selects i=3i = 3i=3 and adds the value aia_iai​ to (i−1)(i-1)(i−1)-th element: a=[10,8]a = [10, 8]a=[10,8].
  • Polycarp selects i=2i = 2i=2 and adds the value aia_iai​ to (i−1)(i-1)(i−1)-th element: a=[18]a = [18]a=[18].

Note that Polycarp could stop performing operations at any time.

Polycarp wondered how many minimum operations he would need to perform to make all the elements of aaa equal (i.e., he wants all aia_iai​ are equal to each other).

Input

The first line contains a single integer ttt (1≤t≤30001 \leq t \leq 30001≤t≤3000) — the number of test cases in the test. Then ttt test cases follow.

The first line of each test case contains a single integer nnn (1≤n≤30001 \leq n \leq 30001≤n≤3000) — the length of the array. The next line contains nnn integers a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ (1≤ai≤1051 \leq a_i \leq 10^51≤ai​≤105) — array aaa.

It is guaranteed that the sum of nnn over all test cases does not exceed 300030003000.

Output

For each test case, output a single number — the minimum number of operations that Polycarp needs to perform so that all elements of the aaa array are the same (equal).
